An open source geospatial trajectory data management \& analysis platform



Mobility\+DB is a database management system for moving object geospatial trajectories, such as G\+PS traces. It adds support for temporal and spatio-\/temporal objects to the \href{https://www.postgresql.org/}{\texttt{ Postgre\+S\+QL}} database and its spatial extension \href{http://postgis.net/}{\texttt{ Post\+G\+IS}}.

Mobility\+DB is developed by the Computer \& Decision Engineering Department of the \href{https://www.ulb.be/}{\texttt{ Université Libre de Bruxelles}} (U\+LB) under the direction of \href{http://cs.ulb.ac.be/members/esteban/}{\texttt{ Prof. Esteban Zimányi}}. U\+LB is an O\+GC Associate Member and member of the O\+GC Moving Feature Standard Working Group (\href{https://www.ogc.org/projects/groups/movfeatswg}{\texttt{ M\+F-\/\+S\+WG}}).

\hypertarget{md_README_autotoc_md17}{}\doxysubsection{Benefits}\label{md_README_autotoc_md17}

\begin{DoxyItemize}
\item Compact geospatial trajectory data storage
\item Rich mobility analytics
\item Big data scale and performance
\item Easy to use full S\+QL interface
\item Compatible with the Postgre\+S\+QL ecosystem
\item Compliant with the \href{https://www.opengeospatial.org/standards/movingfeatures}{\texttt{ Moving Features}} standards from the \href{https://www.opengeospatial.org/}{\texttt{ Open Geospatial Consortium}} (O\+GC).
\item Database adapters to access Mobility\+DB from Python are also available
\begin{DoxyItemize}
\item \href{https://github.com/ULB-CoDE-WIT/MobilityDB-python}{\texttt{ Mobility\+D\+B-\/python}} supports both the \href{https://www.psycopg.org/}{\texttt{ psycopg2}} and the \href{https://github.com/MagicStack/asyncpg}{\texttt{ asyncpg}} adapters for Postgre\+S\+QL and uses the \href{https://github.com/tilery/python-postgis}{\texttt{ postgis}} adapter for Post\+G\+IS. This package is developed by the Mobility\+DB Team.
\item \href{https://github.com/adonmo/mobilitydb-sqlalchemy}{\texttt{ Mobility\+DB S\+Q\+L\+Alchemy}} is another independent package that provides extensions to \href{https://www.sqlalchemy.org/}{\texttt{ S\+Q\+L\+Alchemy}} for interacting with Mobility\+DB.
\end{DoxyItemize}

\hypertarget{md_README_autotoc_md21}{}\doxysubsection{Building \& installation}\label{md_README_autotoc_md21}
Here is the gist\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{git clone https://github.com/MobilityDB/MobilityDB}
\DoxyCodeLine{mkdir MobilityDB/build}
\DoxyCodeLine{cd MobilityDB/build}
\DoxyCodeLine{cmake ..}
\DoxyCodeLine{make}
\DoxyCodeLine{sudo make install}
\DoxyCodeLine{psql -\/c 'CREATE EXTENSION MobilityDB CASCADE'}
\end{DoxyCode}


You should also set the following in postgresql.\+conf\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{shared\_preload\_libraries = 'postgis-\/2.5'}
\DoxyCodeLine{max\_locks\_per\_transaction = 128}
\end{DoxyCode}
\hypertarget{md_README_autotoc_md22}{}\doxysubsection{Docker container}\label{md_README_autotoc_md22}
Docker containers with Mobility\+DB and all its dependencies are available \href{https://github.com/MobilityDB/MobilityDB-docker}{\texttt{ here}}. These images are based on the official \href{https://github.com/docker-library/postgres}{\texttt{ Postgres}} and \href{https://github.com/postgis/docker-postgis}{\texttt{ Postgis}} docker images, please refer to them for more information.

If you have docker installed in your system you can run\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{docker pull codewit/mobilitydb}
\DoxyCodeLine{docker volume create mobilitydb\_data}
\DoxyCodeLine{docker run -\/-\/name "mobilitydb" -\/d -\/p 25432:5432 -\/v mobilitydb\_data:/var/lib/postgresql codewit/mobilitydb}
\end{DoxyCode}


The first command is to download the latest most up-\/to-\/date image of Mobility\+DB. The second command creates a volume container on the host, that we will use to persist the Postgre\+S\+QL database files outside of the Mobility\+DB container. The third command executes this binary image of Postgre\+S\+QL, Post\+G\+IS, and Mobility\+DB with the T\+CP port 5432 in the container mapped to port 25432 on the Docker host (user = pw = docker, db = mobilitydb).\hypertarget{md_README_autotoc_md23}{}\doxysubsection{Issues}\label{md_README_autotoc_md23}
