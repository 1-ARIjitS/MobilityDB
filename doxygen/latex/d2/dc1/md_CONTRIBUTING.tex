If you haven\textquotesingle{}t yet, Chapter 1 of the \href{https://docs.mobilitydb.com/MobilityDB/master/}{\texttt{ manual}} describes how to download and build the sources.\hypertarget{md_CONTRIBUTING_autotoc_md12}{}\doxysection{Reporting Issues and Suggesting Features}\label{md_CONTRIBUTING_autotoc_md12}
When contributing to this repository, please first search the issues to see if your problem/wish has already been reported. If so, add any extra context you might have found, or at least indicate that you too are having the problem/wish. This will help us prioritize common issues.

If your problem is unreported, create a new \href{https://github.com/MobilityDB/MobilityDB/issues}{\texttt{ issue}} for it. This will allow the community to discuss it. When creating an issue, you will be able to choose between multiple templates that we provide to assist in making a clear description of the issue. If you further want to contribute a solution, please go forward and create a pull request.\hypertarget{md_CONTRIBUTING_autotoc_md13}{}\doxysection{Pull Request Process}\label{md_CONTRIBUTING_autotoc_md13}
You can fork this repository, make your own contribution, and submit a pull request. Please put as the pull request title the issue number that it closes (for example, {\ttfamily \mbox{[}Closes issue \#xxx\mbox{]}}). Write a clear log message for your commits. One-\/line messages are fine for small changes, but bigger changes should have more information.\hypertarget{md_CONTRIBUTING_autotoc_md14}{}\doxysection{Contribution Agreement}\label{md_CONTRIBUTING_autotoc_md14}
